## 1. 测试目标与范围

### 1.1 测试目标

本项目是一个面向电商场景的智能导购助手课程项目，测试的核心目标是验证以下三类能力：

1. **功能正确性**：前端页面渲染、用户交互、后端业务逻辑（认证、商品、RAG 问答、客服、订单）均按预期工作。
2. **数据一致性**：数据库（PostgreSQL）、缓存（Redis）、向量库（ChromaDB）与业务层之间的读写一致，特别是在 RAG 检索、商品列表缓存、购物车/订单状态流转中。
3. **稳定性与降级能力**：当 AI 大模型 API 不可用、向量索引为空、Redis 未启动或图片上传失败时，系统能优雅降级，不崩溃、不泄露敏感信息。

### 1.2 测试范围

| 层级 | 范围说明 | 关键被测文件/模块 |
|------|---------|------------------|
| 前端 | Vue 3 + Vite + TypeScript 单页应用 | `frontend/src/components/user/ProductBrowse.vue`、`frontend/src/api.ts` |
| 后端单元 | 不依赖数据库和外部服务的纯函数/业务逻辑 | `backend/services/auth_service.py`、`backend/services/rag_service.py` |
| 后端集成 | Flask Blueprint 路由 + 数据库的真实交互 | `backend/api/auth_routes.py`、`backend/api/merchant_routes.py`、`backend/api/customer_service_routes.py`、`backend/api/routes.py` |
| 端到端/手动 | 浏览器中的完整用户流程与商家后台流程 | 登录、浏览商品、AI 客服对话、下单、商家发货、知识库管理等 |

### 1.3 不在本次文档范围

- 性能压测与容量测试（如并发 1000+ 用户）。
- 安全渗透测试（XSS、SQL 注入专项）。
- 移动端 APP 或小程序测试。
- 第三方爬虫（Scrapy）爬取外部站点的可用性测试。

### 3.2 认证服务测试（auth_service.py）

`backend/services/auth_service.py` 是理想的单元测试对象，因为它只依赖 Python 标准库（`hashlib`、`hmac`、`json`、`base64`、`time`、`secrets`），不依赖数据库。

被测函数签名：

```python
def hash_password(password: str) -> str
def verify_password(password: str, password_hash: str) -> bool
def create_token(user_id: int, username: str, role: str) -> str
def verify_token(token: str) -> dict | None
def require_auth(request) -> tuple[dict | None, dict | None]
def require_merchant(request) -> tuple[dict | None, dict | None]
```

测试用例设计：

1. `hash_password` 返回的字符串格式为 `salt$hash`，且两次调用同一密码生成的结果不同（盐值随机）。
2. `verify_password` 对正确密码返回 `True`，对错误密码返回 `False`。
3. `verify_password` 对非法格式（如不含 `$`、空字符串）返回 `False`，不抛异常。
4. `create_token` 生成的 token 可被 `verify_token` 正确解析，payload 包含 `user_id`、`username`、`role`、`exp`、`iat`。
5. `verify_token` 对过期 token 返回 `None`。
6. `verify_token` 对篡改签名的 token 返回 `None`。
7. `require_auth` 能从请求头 `Authorization: Bearer <token>` 中解析用户信息。
8. `require_merchant` 对普通用户角色返回 `forbidden` 错误。

测试代码示例：

```python
# tests/test_auth_service.py
import time
from unittest.mock import MagicMock

import pytest

from backend.services.auth_service import (
    hash_password,
    verify_password,
    create_token,
    verify_token,
    require_auth,
    require_merchant,
)


def test_hash_password_returns_salt_hash_format():
    hashed = hash_password("password123")
    assert "$" in hashed
    salt, stored = hashed.split("$")
    assert len(salt) == 32  # secrets.token_hex(16) -> 32 个十六进制字符
    assert len(stored) == 64  # sha256 -> 64 个十六进制字符


def test_hash_password_generates_different_salts():
    h1 = hash_password("password123")
    h2 = hash_password("password123")
    assert h1 != h2


def test_verify_password_with_correct_password():
    hashed = hash_password("password123")
    assert verify_password("password123", hashed) is True


def test_verify_password_with_wrong_password():
    hashed = hash_password("password123")
    assert verify_password("wrongpassword", hashed) is False


def test_verify_password_with_invalid_format():
    assert verify_password("password123", "not_a_valid_hash") is False
    assert verify_password("password123", "") is False


def test_create_and_verify_token():
    token = create_token(user_id=1, username="user", role="user")
    payload = verify_token(token)
    assert payload is not None
    assert payload["user_id"] == 1
    assert payload["username"] == "user"
    assert payload["role"] == "user"
    assert "exp" in payload
    assert "iat" in payload


def test_verify_token_expired():
    # 伪造一个已经过期的 token：直接调用内部辅助函数生成
    from backend.services.auth_service import _base64_encode
    import json

    expired_payload = {
        "user_id": 1,
        "username": "user",
        "role": "user",
        "exp": int(time.time()) - 3600,
        "iat": int(time.time()) - 7200,
    }
    from backend.services.auth_service import _sign
    payload_b64 = _base64_encode(json.dumps(expired_payload, separators=(",", ":")))
    signature = _sign(payload_b64)
    expired_token = f"{payload_b64}.{signature}"

    assert verify_token(expired_token) is None


def test_verify_token_tampered_signature():
    token = create_token(user_id=1, username="user", role="user")
    payload_b64, _ = token.split(".")
    tampered_token = f"{payload_b64}.fakesignature"
    assert verify_token(tampered_token) is None


def test_require_auth_with_valid_bearer_token():
    token = create_token(user_id=1, username="user", role="user")
    request = MagicMock()
    request.headers = {"Authorization": f"Bearer {token}"}
    user, error = require_auth(request)
    assert error is None
    assert user["username"] == "user"


def test_require_auth_without_token():
    request = MagicMock()
    request.headers = {}
    user, error = require_auth(request)
    assert user is None
    assert error == {"error": "unauthorized", "message": "请先登录"}


def test_require_merchant_with_user_role():
    token = create_token(user_id=1, username="user", role="user")
    request = MagicMock()
    request.headers = {"Authorization": f"Bearer {token}"}
    user, error = require_merchant(request)
    assert user is None
    assert error == {"error": "forbidden", "message": "需要商家权限"}
```

### 3.3 RAG 服务测试（rag_service.py）

`backend/services/rag_service.py` 中的大量逻辑（问题分类、价格提取、规格约束、关键词提取、无数据库时的回答构建）可以在不连接数据库的情况下进行单元测试。

被测方法：

```python
def _detect_question_type(self, question: str) -> str
def _detect_category(self, question: str) -> Optional[str]
def _extract_price_constraint(self, question: str) -> Optional[float]
def _extract_price_range(self, question: str) -> Optional[tuple[float, float]]
def _extract_spec_constraints(self, question: str) -> list[str]
def _extract_primary_keywords(self, question: str) -> list[str]
def _build_no_match_answer(self, question: str, q_type: str) -> str
```

测试用例设计：

1. 问题类型检测：价格、推荐、品牌、评价、比较、尺寸、功能、售后、通用。
2. 分类检测：输入「猫粮」应识别为「宠物用品」，输入「口红」应识别为「化妆品」。
3. 价格约束：
   - 「300元以内」→ `300.0`
   - 「预算500」→ `500.0`
   - 「200-500元」→ `200.0`（取下限）
   - 「低于300」→ `300.0`
4. 价格范围：
   - 「300元左右」→ `(210.0, 390.0)`（中心值 ±30%）
   - 「200-400元」→ `(200.0, 400.0)`
5. 规格约束：
   - 「0-6个月」→ 包含 `1段`、`0-6`、`新生儿` 等。
   - 「XL码」→ 包含 `XL`。
   - 「老人用的」→ 包含 `老人`、`老年`、`长辈` 等。
6. 核心关键词：输入「推荐一款200元以内的蓝牙耳机」应提取出「蓝牙耳机」。
7. 无匹配回答：输入「你好」应返回问候语；输入未匹配商品的问题应返回引导语。

### 6.4 常见问题

#### Q1：运行 `npm run build` 时报 TypeScript 类型错误

可能原因：

- 某些 Vue 组件使用了未声明的类型。
- `api.ts` 中的接口与实际后端返回字段不匹配。
- 缺少类型定义文件。

排查步骤：

```powershell
# 查看完整错误信息
npm run build 2>&1 | Tee-Object build.log
```

建议先在 `tsconfig.json` 中确认 `strict` 级别，逐步修复错误，不要通过关闭严格模式来绕过。

#### Q2：pytest 运行时报 `ModuleNotFoundError: No module named 'backend'`

解决方法：确保在 `pyproject.toml` 中已配置 `pythonpath = ["backend"]`，并且从项目根目录运行 pytest：

```powershell
cd C:\Users\Jin\Desktop\ecommerce-ai-copy-guide-main
python -m pytest tests -v
```

不要直接在 `tests` 目录下运行 `pytest`。

#### Q3：集成测试连接真实 PostgreSQL 失败

解决方法：

1. 检查 `.env` 中 `DATABASE_URL` 是否正确。
2. 确认 PostgreSQL 服务已启动。
3. 数据库和用户已创建：

```sql
CREATE DATABASE ecommerce_ai;
CREATE USER ecommerce_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE ecommerce_ai TO ecommerce_user;
```

#### Q4：AI 客服接口返回 500

可能原因：

- 未配置 `AI_API_KEY` 但代码路径未正确降级。
- ChromaDB 向量索引初始化失败。
- `openai` 库版本不兼容。

排查方法：查看后端日志，确认 `rag_service.answer_question` 中的异常捕获是否生效。临时可将 `AI_API_KEY` 置空，强制走模板回答分支。

#### Q5：前端请求后端报 CORS 错误

`backend/app.py` 中已经配置了全局 CORS：

```python
@app.after_request
def add_cors_headers(response):
    response.headers["Access-Control-Allow-Origin"] = "*"
    response.headers["Access-Control-Allow-Headers"] = "Content-Type,Authorization"
    response.headers["Access-Control-Allow-Methods"] = "GET,POST,PUT,DELETE,OPTIONS"
    return response
```

如果仍报错，检查：

1. 前端 `api.ts` 中的 `API_BASE_URL` 是否指向正确后端地址。
2. 浏览器是否使用了缓存的旧配置。
3. 是否有代理或防火墙拦截。

#### Q6：图片/头像上传后无法访问

确认：

1. `backend/uploads` 目录存在且后端有写入权限。
2. 访问上传文件时使用路径 `/uploads/avatars/xxx.jpg`。
3. `backend/app.py` 中已注册 `send_from_directory` 路由：

```python
@app.get("/uploads/<path:filename>")
def serve_upload(filename: str):
    upload_folder = app.config.get("UPLOAD_FOLDER") or os.path.join(
        os.path.dirname(__file__), "uploads"
    )
    return send_from_directory(upload_folder, filename)
```

```python
# tests/test_aftersale_fallback.py
import requests

def test_aftersale_fallback(client, normal_user):
    token = create_token(normal_user.id, normal_user.username, normal_user.role)
    questions = ["如何退换货", "怎么退货", "换货流程", "售后保修"]
    for q in questions:
        response = client.post(
            "/api/user/qa/ask",
            headers={"Authorization": f"Bearer {token}"},
            json={"question": q},
        )
        assert response.status_code == 200
        data = response.get_json()
        assert "平台售后政策" in data["answer"]
        assert data.get("related_products") == [] or not data.get("related_products")
```

商家回复接口的自动化回归：

```python
def test_merchant_reply(client, merchant_user, normal_user):
    # 先由用户发送一条消息
    user_token = create_token(normal_user.id, normal_user.username, normal_user.role)
    client.post(
        "/api/cs/messages",
        headers={"Authorization": f"Bearer {user_token}"},
        json={"content": "请问怎么退货"},
    )

    # 商家回复
    merchant_token = create_token(merchant_user.id, merchant_user.username, merchant_user.role)
    response = client.post(
        f"/api/cs/merchant/threads/{normal_user.id}/reply",
        headers={"Authorization": f"Bearer {merchant_token}"},
        json={"content": "支持 7 天无理由退货"},
    )
    assert response.status_code == 201
    data = response.get_json()
    assert data["data"]["sender_role"] == "merchant"
    assert data["data"]["sender_id"] == merchant_user.id
```
