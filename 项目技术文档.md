# 电商 AI 导购助手 — 项目技术文档

## 1. 项目概述

本项目是一个面向电商场景的 AI 导购与运营管理平台，包含两大核心端：

- **用户前台**：商品浏览、购物车/订单、智能问答（RAG）、评论提交、收藏/地址管理等。
- **商家后台**：商品 CRUD、知识库管理、订单发货/退换货处理、用户管理、收入看板、AI 文案/直播脚本/评论分析工具。

后端采用 Flask + SQLAlchemy 构建 REST API，前端使用 Vue 3 + Vite + TypeScript 实现单页应用（SPA），通过 `fetch` 与后端交互。AI 能力支持 DeepSeek / OpenAI / Mock 三种提供者切换，RAG 检索使用 ChromaDB + 硅基流动 Embedding API 实现双阶段混合检索。

核心交互流程可概括为：

1. 用户在浏览器中操作 Vue 组件；
2. `api.ts` 将组件状态转换为 HTTP 请求，并携带 `Authorization: Bearer <token>` 头部；
3. Flask `app.py` 注册多个 Blueprint，根据路径前缀分发到对应路由模块；
4. 路由处理函数通过 `backend.database.SessionLocal` 开启事务，执行 SQLAlchemy 查询或写入；
5. 需要 AI 能力时，调用 `rag_service.py` 或 `ai_provider.py`，后者再调用外部 LLM / Embedding API；
6. 结果序列化为 JSON 返回前端，Vue 组件更新视图。

## 4. 请求处理流程

### 4.1 前端视图路由与状态管理

前端不是使用 `vue-router`，而是在 `App.vue` 中通过响应式状态 `currentView` 实现条件渲染：

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import Login from './components/Login.vue'
import MerchantLayout from './components/merchant/MerchantLayout.vue'
import UserLayout from './components/user/UserLayout.vue'
// ... 其他组件

type AppView = 'login' | 'merchant' | 'user'

interface UserInfo {
  username: string
  role: 'user' | 'merchant'
  nickname?: string
  token?: string
  [key: string]: unknown
}

const currentView = ref<AppView>('login')
const userInfo = ref<UserInfo | null>(null)

// 应用启动时检查本地登录态，实现刷新保持登录
onMounted(() => {
  const stored = localStorage.getItem('userInfo')
  if (stored) {
    try {
      const info = JSON.parse(stored) as UserInfo
      if (info && info.username && info.role) {
        userInfo.value = info
        currentView.value = info.role === 'merchant' ? 'merchant' : 'user'
      } else {
        throw new Error('无效的用户信息')
      }
    } catch {
      localStorage.removeItem('userInfo')
      localStorage.removeItem('token')
      currentView.value = 'login'
    }
  }
})

// 登录成功：根据角色分流
function handleLoginSuccess(info: UserInfo) {
  userInfo.value = info
  currentView.value = info.role === 'merchant' ? 'merchant' : 'user'
}
</script>

<template>
  <Login v-if="currentView === 'login'" @login-success="handleLoginSuccess" />
  <MerchantLayout v-else-if="currentView === 'merchant' && userInfo" :user-info="userInfo" @logout="handleLogout">
    <template #default="{ page }">
      <RevenueDashboard v-if="page === 'dashboard'" />
      <ProductManage v-else-if="page === 'products'" />
      <!-- ... 其他商家页面 -->
    </template>
  </MerchantLayout>
  <UserLayout v-else-if="currentView === 'user' && userInfo" :user-info="userInfo" @logout="handleLogout">
    <template #default="{ page }">
      <ProductBrowse v-if="page === 'products'" />
      <UserProfile v-else-if="page === 'profile'" />
    </template>
  </UserLayout>
</template>
```

`MerchantLayout` 和 `UserLayout` 通过 slot 将当前子页面名称 `page` 注入进来，`App.vue` 再根据 `page` 的值渲染对应功能组件。登录成功后，`Login.vue` 会触发 `login-success` 事件，`App.vue` 把用户信息写入 `localStorage` 并切换视图；刷新页面时通过 `onMounted` 恢复登录态。

### 4.2 前端 API 封装

所有后端请求集中在 `frontend/src/api.ts` 中，使用原生 `fetch` 封装：

```typescript
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000'

async function handleResponse<T>(response: Response): Promise<T> {
  if (!response.ok) {
    const error = await response.json().catch(() => ({ message: '请求失败' }))
    throw new Error(error.message || `HTTP ${response.status}`)
  }
  return response.json()
}

export const api = {
  async generateCopy(data: CopyGenerationRequest): Promise<CopyGenerationResponse> {
    const response = await fetch(`${API_BASE_URL}/api/copy/generate`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    })
    return handleResponse<CopyGenerationResponse>(response)
  },

  async recommendGuide(data: GuideRecommendationRequest): Promise<GuideRecommendationResponse> {
    const response = await fetch(`${API_BASE_URL}/api/guide/recommend`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    })
    return handleResponse<GuideRecommendationResponse>(response)
  },

  // ... 其他接口
}
```

注意：该文件主要封装了 AI 工具类接口（文案生成、推荐、评论分析、直播脚本等），但购物车、订单、用户地址等接口分散在各自 Vue 组件中直接调用。Base URL 通过 `import.meta.env.VITE_API_BASE_URL` 注入，默认指向 `http://localhost:8000`。

### 4.4 路由处理函数示例

以商家商品创建和用户加入购物车为例，展示从 HTTP 请求到数据库写入的完整链路。

#### 商家创建商品

`backend/api/merchant_routes.py`：

```python
from flask import Blueprint, jsonify, request
from sqlalchemy import select
from backend.database import SessionLocal
from backend.models.product import Product
from backend.services.auth_service import require_merchant
from backend.services.rag_service import RAGService

merchant_bp = Blueprint("merchant", __name__)
rag_service = RAGService()

@merchant_bp.post("/merchant/products")
def create_product():
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401

    data = request.get_json(silent=True) or {}
    required = ["name"]
    for field in required:
        if not data.get(field):
            return jsonify({"error": "invalid_input", "message": f"缺少必填字段: {field}"}), 400

    image_urls = data.get("image_urls") or []
    image_url = data.get("image_url", "")
    if not image_url and image_urls:
        image_url = image_urls[0]

    with SessionLocal() as db:
        product = Product(
            name=data["name"],
            category=data.get("category", ""),
            price=float(data.get("price", 0) or 0),
            specs=data.get("specs", ""),
            selling_points=data.get("selling_points", ""),
            image_url=image_url,
            image_urls=image_urls if image_urls else None,
            brand=data.get("brand", ""),
            is_published=bool(data.get("is_published", True)),
        )
        db.add(product)
        product.display_id = generate_product_display_id(db)
        db.commit()
        db.refresh(product)

        try:
            rag_service.auto_build_from_product(product)
        except Exception as e:
            logger.warning(f"知识库自动构建失败: {e}")

        return jsonify({"message": "商品创建成功", "product": product.to_dict()}), 201
```

流程说明：

1. `require_merchant(request)` 从请求头提取 JWT，验证用户身份和商家角色；
2. 校验必填字段 `name`；
3. 使用 `SessionLocal()` 开启数据库会话；
4. 创建 `Product` 模型实例并写入；
5. 调用 `rag_service.auto_build_from_product(product)` 自动生成规格、卖点、售后等知识库条目；
6. 提交事务并返回 JSON。

#### 用户加入购物车

`backend/api/user_routes.py`：

```python
from flask import Blueprint, jsonify, request
from sqlalchemy import select
from backend.database import SessionLocal
from backend.models.product import Product
from backend.models.shopping import CartItem
from backend.services.auth_service import require_auth

user_bp = Blueprint("user", __name__)

@user_bp.post("/user/cart")
def add_to_cart():
    user_payload, error = require_auth(request)
    if error:
        return jsonify(error), 401

    data = request.get_json(silent=True) or {}
    product_id = _safe_int(data.get("product_id"))
    quantity = _safe_int(data.get("quantity"), 1) or 1

    if not product_id:
        return jsonify({"error": "invalid_input", "message": "缺少商品ID"}), 400

    with SessionLocal() as db:
        product = db.get(Product, product_id)
        if not product:
            return jsonify({"error": "not_found", "message": "商品不存在"}), 404

        existing = db.execute(
            select(CartItem).where(
                CartItem.user_id == user_payload["user_id"],
                CartItem.product_id == product_id,
            )
        ).scalar_one_or_none()

        if existing:
            existing.quantity += quantity
            db.commit()
            return jsonify({"message": "数量已更新", "item": existing.to_dict()})
        else:
            item = CartItem(
                user_id=user_payload["user_id"],
                product_id=product_id,
                quantity=quantity,
            )
            db.add(item)
            db.commit()
            return jsonify({"message": "已加入购物车", "item": item.to_dict()}), 201
```

这里同样先通过 `require_auth` 校验登录态，再查询商品与购物车记录，执行新增或更新操作。

### 4.5 数据库连接与事务管理

`backend/database.py` 负责数据库引擎、会话与初始化：

```python
from sqlalchemy import create_engine, event
from sqlalchemy.orm import DeclarativeBase, sessionmaker, Session

def _resolve_database_url() -> str:
    configured_url = os.getenv("DATABASE_URL", "")
    if not configured_url:
        return "sqlite:///./ecommerce_ai.db"
    if configured_url.startswith("sqlite"):
        return configured_url
    if configured_url.startswith("postgresql"):
        try:
            import psycopg2
            engine = create_engine(configured_url, pool_pre_ping=True)
            with engine.connect() as conn:
                conn.execute(__import__("sqlalchemy").text("SELECT 1"))
            return configured_url
        except Exception:
            return "sqlite:///./ecommerce_ai.db"
    return configured_url

DATABASE_URL = _resolve_database_url()

if DATABASE_URL.startswith("sqlite"):
    engine = create_engine(
        DATABASE_URL,
        echo=False,
        connect_args={"check_same_thread": False},
    )
    @event.listens_for(engine, "connect")
    def set_sqlite_pragma(dbapi_connection, connection_record):
        cursor = dbapi_connection.cursor()
        cursor.execute("PRAGMA foreign_keys=ON")
        cursor.close()
else:
    engine = create_engine(
        DATABASE_URL,
        pool_size=10,
        max_overflow=20,
        pool_pre_ping=True,
        echo=os.getenv("APP_ENV", "development") == "development",
    )

SessionLocal = sessionmaker(bind=engine)

class Base(DeclarativeBase):
    pass

def get_db() -> Session:
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

关键设计：

- **自动降级**：当 `DATABASE_URL` 未配置或 PostgreSQL 不可用时，自动回退到 SQLite，便于课程演示与本地开发。
- **外键约束**：SQLite 默认不启用外键，通过 `event.listen_for(engine, "connect")` 强制开启 `PRAGMA foreign_keys=ON`。
- **会话管理**：路由中直接使用 `with SessionLocal() as db:` 管理会话，异常时由 SQLAlchemy 自动回滚（默认行为），成功时显式 `commit()`。
- **兼容性迁移**：`init_db()` 在启动时自动检测并补齐老表缺失字段（如 `display_id`、`image_urls`、`is_published` 等），降低 schema 变更成本。

## 5. AI / RAG 流程

### 5.1 整体流程

RAG（检索增强生成）用于智能导购问答。当用户在商品页或问答页输入问题时，系统执行以下步骤：

1. **问题理解**：提取价格约束、规格约束、分类意图、问题类型（价格/推荐/品牌/功能/尺寸/评价等）。
2. **第一阶段：规则过滤**：使用 SQLAlchemy `ilike` 查询商品名，结合分类、价格、规格等规则过滤，得到候选商品列表。
3. **第二阶段：向量重排序**：调用 `vector_index.search()` 对候选集做语义相似度排序，选出最匹配商品。
4. **数据组装**：读取商品详情、真实评论、同类推荐商品。
5. **AI 生成**：调用 LLM（DeepSeek / OpenAI）生成自然语言回答；无 API Key 时降级为模板回答。
6. **流式输出**：通过 SSE 将商品卡片、推荐列表与 AI 回答逐 token 推送给前端。

核心入口在 `backend/services/rag_service.py`：


### 5.2 向量检索服务

`backend/services/vector_index.py` 基于 **ChromaDB** + **硅基流动 Embedding API**：

```python
import chromadb
import requests

_CHROMA_DIR = os.path.join(
    os.path.dirname(os.path.dirname(os.path.dirname(__file__))),
    "chroma_db",
)
_COLLECTION_NAME = "products"
_SILICONFLOW_API_KEY = os.getenv("SILICONFLOW_API_KEY", "sk-lgdbbjgswowlmgzmcklglxowqphzmjfgfplybvmosoftycmy")
_SILICONFLOW_BASE_URL = "https://api.siliconflow.cn/v1"
_EMBEDDING_MODEL = "BAAI/bge-large-zh-v1.5"

class VectorIndex:
    _instance: Optional["VectorIndex"] = None
    _lock = threading.Lock()

    def __new__(cls):
        if cls._instance is None:
            with cls._lock:
                if cls._instance is None:
                    cls._instance = super().__new__(cls)
                    cls._instance._initialized = False
        return cls._instance

    def _ensure_client(self):
        if self._client is None:
            os.makedirs(_CHROMA_DIR, exist_ok=True)
            self._client = chromadb.PersistentClient(path=_CHROMA_DIR)
            self._collection = self._client.get_or_create_collection(
                name=_COLLECTION_NAME,
                metadata={"hnsw:space": "cosine"},
            )

    def _embed_batch(self, texts: list[str]) -> list[list[float]]:
        all_embeddings = []
        batch_size = 32
        for i in range(0, len(texts), batch_size):
            batch = texts[i:i + batch_size]
            response = requests.post(
                f"{_SILICONFLOW_BASE_URL}/embeddings",
                headers={
                    "Authorization": f"Bearer {_SILICONFLOW_API_KEY}",
                    "Content-Type": "application/json",
                },
                json={
                    "model": _EMBEDDING_MODEL,
                    "input": batch,
                    "encoding_format": "float",
                },
                proxies={"http": None, "https": None},
                timeout=60,
            )
            response.raise_for_status()
            data = response.json()
            embeddings = [None] * len(batch)
            for item in data["data"]:
                embeddings[item["index"]] = item["embedding"]
            all_embeddings.extend(embeddings)
        return all_embeddings

    def build_index(self, products: list) -> int:
        self._ensure_client()
        ids, documents, metadatas = [], [], []
        for p in products:
            doc_parts = [p.name or ""]
            if p.selling_points:
                doc_parts.append(p.selling_points)
            if p.category:
                doc_parts.append(p.category)
            if p.brand:
                doc_parts.append(p.brand)
            doc = " | ".join(doc_parts)
            ids.append(str(p.id))
            documents.append(doc)
            metadatas.append({
                "product_id": p.id,
                "name": p.name or "",
                "category": p.category or "",
                "price": float(p.price or 0),
                "brand": p.brand or "",
            })
        embeddings = self._embed_batch(documents)
        for i in range(0, len(ids), 500):
            self._collection.upsert(
                ids=ids[i:i+500],
                documents=documents[i:i+500],
                embeddings=embeddings[i:i+500],
                metadatas=metadatas[i:i+500],
            )
        return self._collection.count()

    def search(self, query: str, candidate_ids: Optional[list[int]] = None, top_k: int = 10) -> list[dict]:
        self._ensure_client()
        query_embedding = self._embed_query(query)
        where_filter = None
        if candidate_ids is not None and len(candidate_ids) > 0:
            where_filter = {"product_id": {"$in": [int(pid) for pid in candidate_ids]}}
        query_params = {
            "query_embeddings": [query_embedding],
            "n_results": min(top_k, len(candidate_ids)) if candidate_ids else top_k,
            "include": ["metadatas", "distances"],
        }
        if where_filter:
            query_params["where"] = where_filter
        results = self._collection.query(**query_params)
        output = []
        if results and results.get("ids"):
            for idx in range(len(results["ids"][0])):
                meta = results["metadatas"][0][idx]
                dist = results["distances"][0][idx]
                similarity = max(0.0, 1.0 - dist / 2.0)
                output.append({
                    "product_id": meta.get("product_id"),
                    "name": meta.get("name", ""),
                    "category": meta.get("category", ""),
                    "price": meta.get("price", 0),
                    "brand": meta.get("brand", ""),
                    "similarity": round(similarity, 4),
                })
        return output
```

说明：

- `VectorIndex` 使用单例模式，避免重复初始化 ChromaDB。
- Embedding 通过硅基流动在线 API 批量生成，每批 32 条，模型为 `BAAI/bge-large-zh-v1.5`。
- 向量索引构建时会将商品名称、卖点、分类、品牌拼接为文档文本，生成 Embedding 后写入 ChromaDB；`upsert` 模式支持重复构建时自动覆盖。
- `search` 方法支持传入 `candidate_ids`，仅在规则过滤后的候选集中做语义排序，实现“双阶段混合检索”。
- 余弦距离通过 `1 - dist / 2` 转换为相似度分数。

### 5.3 RAG 两阶段混合检索

`rag_service.py` 中的 `_search_product_by_question` 是检索核心：

```python

    # 1. 提取约束条件
    price_limit = self._extract_price_constraint(question)
    price_range = self._extract_price_range(question) if not price_limit else None
    spec_constraints = self._extract_spec_constraints(question)

    # 2. 精确商品名匹配（最高优先级）
    exact_match = self._try_exact_product_match(db, question)
    if exact_match:
        return exact_match

    primary_kws = self._extract_primary_keywords(question)
    detected_category = self._detect_category(question)

    if primary_kws:
        conditions = [Product.name.ilike(f"%{kw}%") for kw in primary_kws]
        if detected_category:
            results = list(db.execute(
                select(Product).where(
                    Product.category == detected_category,
                    or_(*conditions),
                ).limit(50)
            ).scalars().all())
        else:
            results = list(db.execute(
                select(Product).where(or_(*conditions)).limit(50)
            ).scalars().all())

        # 应用价格、规格约束
        results = self._apply_price_filter(results, price_limit, price_range)
        results = self._apply_spec_filter(results, spec_constraints)

        if results:
            best = self._vector_rerank(question, results, primary_kws, spec_constraints)
            if best:
                return best

    # 4. 通用关键词搜索 / 全局搜索 ...
```

`_vector_rerank` 综合向量相似度、关键词匹配度和商品评分进行排序：

```python
def _vector_rerank(self, question, candidates, keywords, spec_constraints, detected_category=None):
    if not candidates:
        return None
    try:
        candidate_ids = [p.id for p in candidates]
        query_text = question
        if keywords:
            query_text = question + " " + " ".join(keywords[:3])
        vector_results = vector_index.search(
            query=query_text,
            candidate_ids=candidate_ids,
            top_k=len(candidates),
        )
        if vector_results:
            sim_map = {r["product_id"]: r["similarity"] for r in vector_results}

            def combined_score(p: Product) -> float:
                vec_sim = sim_map.get(p.id, 0.0)
                kw_score = 0.0
                name_lower = (p.name or "").lower()
                for kw in keywords:
                    if kw.lower() in name_lower:
                        kw_score += len(kw) * 2.0
                for sc in spec_constraints:
                    if sc.lower() in name_lower:
                        kw_score += 10.0
                if detected_category and p.category == detected_category:
                    kw_score += 5.0
                rating_score = (p.rating or 0) * 0.1
                return vec_sim * 0.6 + min(kw_score, 30.0) * 0.3 + rating_score * 0.1

            candidates.sort(key=combined_score, reverse=True)
            return candidates[0]
    except Exception as e:
        logger.warning(f"向量重排序失败，降级为关键词排序: {e}")

    # 降级：纯关键词加权排序
    def fallback_score(p: Product) -> float:
        ...
    candidates.sort(key=fallback_score, reverse=True)
    return candidates[0]
```

综合评分公式为：`向量相似度 × 0.6 + 关键词匹配度 × 0.3 + 商品评分 × 0.1`。当向量服务不可用时，自动降级为纯关键词排序，保证系统可用性。

### 6.1 JWT 创建与校验

项目没有引入 `PyJWT` 等第三方库，而是基于 Python 标准库实现了简化版 JWT：

```python
# backend/services/auth_service.py
import hashlib
import hmac
import json
import os
import time
import secrets

JWT_SECRET = os.getenv("JWT_SECRET", "ecommerce-ai-secret-key-2026")
JWT_EXPIRE_HOURS = 24


def hash_password(password: str) -> str:
    """密码哈希 - 使用 salt + sha256"""
    salt = secrets.token_hex(16)
    hashed = hashlib.sha256(f"{salt}{password}".encode()).hexdigest()
    return f"{salt}${hashed}"


def verify_password(password: str, password_hash: str) -> bool:
    try:
        salt, stored_hash = password_hash.split("$")
        computed = hashlib.sha256(f"{salt}{password}".encode()).hexdigest()
        return hmac.compare_digest(computed, stored_hash)
    except (ValueError, AttributeError):
        return False


def create_token(user_id: int, username: str, role: str) -> str:
    """创建 JWT Token (简化版)"""
    payload = {
        "user_id": user_id,
        "username": username,
        "role": role,
        "exp": int(time.time()) + JWT_EXPIRE_HOURS * 3600,
        "iat": int(time.time()),
    }
    payload_json = json.dumps(payload, separators=(",", ":"))
    payload_b64 = _base64_encode(payload_json)
    signature = _sign(f"{payload_b64}")
    return f"{payload_b64}.{signature}"


def verify_token(token: str) -> dict | None:
    """验证 Token，返回 payload 或 None"""
    try:
        parts = token.split(".")
        if len(parts) != 2:
            return None
        payload_b64, signature = parts
        expected_sig = _sign(payload_b64)
        if not hmac.compare_digest(signature, expected_sig):
            return None
        payload_json = _base64_decode(payload_b64)
        payload = json.loads(payload_json)
        if payload.get("exp", 0) < time.time():
            return None
        return payload
    except Exception as e:
        logger.debug(f"Token 验证失败: {e}")
        return None


def _base64_encode(data: str) -> str:
    import base64
    return base64.urlsafe_b64encode(data.encode()).decode().rstrip("=")


def _base64_decode(data: str) -> str:
    import base64
    padding = 4 - len(data) % 4
    if padding != 4:
        data += "=" * padding
    return base64.urlsafe_b64decode(data.encode()).decode()


def _sign(data: str) -> str:
    return hmac.new(JWT_SECRET.encode(), data.encode(), hashlib.sha256).hexdigest()
```

说明：

- Token 结构为 `base64url(payload).signature`，省略了标准 JWT 的 header 部分。
- Payload 包含 `user_id`、`username`、`role`、`exp`（过期时间）和 `iat`（签发时间）。
- 签名使用 HMAC-SHA256，密钥来自环境变量 `JWT_SECRET`。
- 密码使用随机 salt + SHA256，并通过 `hmac.compare_digest` 防止时序攻击。

> **安全提示**：当前 `JWT_SECRET` 存在硬编码默认值，生产环境务必通过环境变量替换为强随机密钥；同时建议将 SHA256 升级为 `bcrypt` / `argon2` 等专门密码哈希算法。

### 6.2 登录与注册

`backend/api/auth_routes.py` 实现登录、注册与当前用户信息接口：

```python
@auth_bp.post("/auth/login")
def login():
    data = request.get_json(silent=True) or {}
    username = data.get("username", "").strip()
    password = data.get("password", "")
    role = data.get("role", "user")

    if not username or not password:
        return jsonify({"error": "invalid_input", "message": "账号和密码不能为空"}), 400

    with SessionLocal() as db:
        user = db.execute(
            select(User).where(User.username == username)
        ).scalar_one_or_none()

        if not user:
            return jsonify({"error": "not_found", "message": "账号不存在"}), 404
        if not verify_password(password, user.password_hash):
            return jsonify({"error": "wrong_password", "message": "密码错误"}), 401
        if not user.is_active:
            return jsonify({"error": "disabled", "message": "账号已被禁用"}), 403
        if role and role != user.role:
            return jsonify({
                "error": "role_mismatch",
                "message": f"该账号是{'商家管理员' if user.role == 'merchant' else '普通用户'}，请切换身份登录"
            }), 403

        token = create_token(user.id, user.username, user.role)
        user_data = user.to_dict()
        user_data["token"] = token
        return jsonify({"message": "登录成功", "user": user_data, "token": token})
```

注册逻辑类似：校验用户名/密码长度，检查用户名是否已存在，创建用户后签发 token。登录成功后，前端将 `userInfo` 与 `token` 存入 `localStorage`，后续请求通过 `Authorization: Bearer <token>` 头部携带。

### 6.3 请求认证装饰

`auth_service.py` 提供三个辅助函数用于从请求中提取并校验身份：

```python
def get_user_from_request(request) -> dict | None:
    auth_header = request.headers.get("Authorization", "")
    if auth_header.startswith("Bearer "):
        token = auth_header[7:]
        return verify_token(token)
    return None


def require_auth(request) -> tuple[dict | None, dict | None]:
    user = get_user_from_request(request)
    if not user:
        return None, {"error": "unauthorized", "message": "请先登录"}
    return user, None


def require_merchant(request) -> tuple[dict | None, dict | None]:
    user, error = require_auth(request)
    if error:
        return None, error
    if user.get("role") != "merchant":
        return None, {"error": "forbidden", "message": "需要商家权限"}
    return user, None
```

所有需要登录的接口都会在路由函数开头调用 `require_auth` 或 `require_merchant`，例如：

```python
@merchant_bp.get("/merchant/products")
def list_merchant_products():
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401
    ...

@user_bp.get("/user/cart")
def get_cart():
    user_payload, error = require_auth(request)
    if error:
        return jsonify(error), 401
    ...
```

## 7. 关键设计决策说明

### 7.1 单页应用 + 状态驱动视图

前端没有引入 `vue-router`，而是在 `App.vue` 中通过 `currentView`（`login` / `merchant` / `user`）和 `MerchantLayout` / `UserLayout` 的 `page` slot prop 实现页面切换。这样做的好处是：

- 项目规模不大，减少依赖；
- 登录态与角色分流逻辑集中，刷新页面时可通过 `localStorage` 快速恢复；
- 商家后台与用户前台共用同一套顶层状态。

缺点是在 URL 上不会体现当前页面，无法直接通过浏览器地址栏进入某个子页面，也不支持浏览器前进/后退。后续若页面继续增加，建议迁移到 `vue-router`。

### 7.2 SQLite 自动降级与零配置开发

`database.py` 中的 `_resolve_database_url()` 在未配置 `DATABASE_URL` 或 PostgreSQL 不可用时自动回退到 SQLite，并启用外键约束。这使得课程作业或本地演示无需安装 PostgreSQL 即可运行，同时通过 `pool_pre_ping`、`pool_size` 等参数保留生产级 PostgreSQL 能力。

### 7.3 兼容性迁移脚本

`init_db()` 中内置了大量 `_add_*_column()` 和 `_hash_existing_plain_passwords()`、`_normalize_order_numbers()` 等函数，在应用启动时自动：

- 补齐老表缺失字段（如 `display_id`、`image_urls`、`videos`、`is_published`、`sender_id` 等）；
- 将残留明文密码哈希化并清空；
- 规范化订单编号格式。

这种“启动时自动迁移”的方式适合课程项目快速迭代，但在生产环境中建议迁移到 Alembic 等正式迁移工具。

### 7.4 AI 提供者可插拔 + Mock 降级

通过 `AIProvider` 抽象接口与 `get_ai_provider()` 工厂函数，系统支持三种模式：

- `AI_PROVIDER=deepseek`：默认，调用 DeepSeek API；
- `AI_PROVIDER=openai`：调用 OpenAI API；
- `AI_PROVIDER=mock` 或未配置 `AI_API_KEY`：使用 `MockAIService`。

Mock 实现覆盖文案生成、推荐、问答、评论分析、直播脚本五大功能，确保项目在没有外部 API Key 时仍可完整运行和演示。

### 7.5 双阶段混合 RAG

RAG 检索采用“规则过滤 + 向量重排序”两阶段策略：

- 第一阶段使用 SQL 模糊查询 + 分类/价格/规格约束，快速缩小候选范围，可解释、可控；
- 第二阶段使用 Embedding 语义相似度对候选集重排序，提升语义理解能力；
- 向量服务不可用时自动降级为关键词加权排序，保证可用性。

这种设计兼顾了检索效率、结果可控性与语义理解能力。

### 7.6 简化版 JWT

项目使用标准库实现 JWT，避免额外依赖，适合教学场景。Token 仅包含必要字段并设置 24 小时过期时间。但需要注意：

- 默认 `JWT_SECRET` 为硬编码字符串，生产环境必须替换；
- 当前实现未包含标准 JWT 的 header 与算法声明；
- 密码哈希使用 SHA256 + salt，建议生产环境改用 `bcrypt` 或 `argon2`。

### 7.7 前后端 CORS 与上传文件服务

`app.py` 通过 `@app.after_request` 全局添加 CORS 头部，允许所有来源访问：

```python
@app.after_request
def add_cors_headers(response):
    response.headers["Access-Control-Allow-Origin"] = "*"
    response.headers["Access-Control-Allow-Headers"] = "Content-Type,Authorization"
    response.headers["Access-Control-Allow-Methods"] = "GET,POST,PUT,DELETE,OPTIONS"
    return response
```

上传文件保存在 `backend/uploads/` 目录，并通过 `/uploads/<path:filename>` 路由提供静态访问。开发环境可直接通过 `http://localhost:8000/uploads/...` 访问上传的图片/视频。

## 9. 结论

本项目是一个功能完整的电商 AI 导购与运营平台示例，技术栈清晰、分层明确：

- **前端**使用 Vue 3 + Vite + TypeScript 构建响应式 SPA；
- **后端**使用 Flask + SQLAlchemy 提供 REST API；
- **数据库**支持 SQLite 开发降级与 PostgreSQL 生产部署；
- **AI 能力**通过可插拔的 Provider 架构支持 DeepSeek、OpenAI 与 Mock 三种模式；
- **RAG 检索**采用规则过滤 + 向量语义重排序，结合真实商品数据与评论生成回答；
- **认证**使用自实现 JWT + salt SHA256 密码哈希，适合教学与演示。

在后续演进中，可考虑引入 `vue-router` 完善前端路由、使用 Alembic 替代启动时迁移脚本、将密码哈希升级为 `bcrypt`、并为 `JWT_SECRET` 强制要求环境变量配置以提升生产安全性。

```python
# backend/services/rag_service.py
```

### 10.2 商家端客服回复 HTTP 500

`require_merchant` 返回的是 JWT payload 字典，原代码直接使用 `merchant.id` 导致 AttributeError。修复后的 `merchant_reply` 先根据 `merchant["user_id"]` 查询 `User` 对象，再用 `merchant_user.id` 写入 `sender_id`：

```python
# backend/api/customer_service_routes.py
merchant_user = db.get(User, merchant["user_id"])
if not merchant_user:
    return jsonify({"error": "not_found", "message": "商家账号不存在"}), 404

msg = CustomerServiceMessage(
    user_id=user_id,
    product_id=product_id,
    sender_id=merchant_user.id,
    sender_role="merchant",
    content=content,
    is_read=True,
)
```

### 10.3 商家端用户删除的外键处理

`users` 表与订单、收藏、地址、评论、客服消息等多张表存在直接或间接关联。直接删除会触发外键约束失败。修复方案在 `delete_user` 中按业务重要性依次清理：

```python
# backend/api/merchant_routes.py
db.execute(delete(CartItem).where(CartItem.user_id == user_id))
db.execute(delete(UserFavorite).where(UserFavorite.user_id == user_id))
db.execute(delete(BrowseHistory).where(BrowseHistory.user_id == user_id))
db.execute(delete(UserAddress).where(UserAddress.user_id == user_id))
db.execute(delete(CustomerServiceMessage).where(CustomerServiceMessage.user_id == user_id))
db.execute(update(QARecord).where(QARecord.user_id == user_id).values(user_id=None))
db.execute(update(Review).where(Review.user_id == user_id).values(user_id=None))

order_ids = [row[0] for row in db.execute(select(Order.id).where(Order.user_id == user_id)).all()]
if order_ids:
    db.execute(delete(OrderItem).where(OrderItem.order_id.in_(order_ids)))
    db.execute(delete(Order).where(Order.id.in_(order_ids)))

db.delete(target_user)
db.commit()
```

该删除操作仍禁止商家删除自己的账号：

```python
if user_id == merchant["user_id"]:
    return jsonify({"error": "forbidden", "message": "不能删除自己的账号"}), 403
```
