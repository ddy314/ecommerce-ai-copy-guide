# 后端设计文档

本文档基于 `ecommerce-ai-copy-guide` 项目的后端源码，对其架构、认证、数据库会话、主要接口流程及错误处理进行说明。涉及文件包括 `backend/app.py`、`backend/database.py`、`backend/api/*.py`、`backend/services/auth_service.py` 以及 `backend/models/*.py`。

---

## 一、后端架构与 Flask Blueprint 组织

后端采用 **Flask** 作为 Web 框架，整体遵循“应用工厂 + Blueprint”模式。入口文件 `backend/app.py` 中的 `create_app()` 负责：

1. 加载配置（`AppConfig.from_env()`）并初始化日志。
2. 创建 `Flask` 实例，配置上传目录。
3. 注册全局错误处理、`/health` 探针、CORS 头、上传文件访问路由。
4. 初始化数据库表并创建默认账号。
5. 注册全部 Blueprint。

```python
# backend/app.py
from backend.api.routes import api_bp
from backend.api.auth_routes import auth_bp
from backend.api.merchant_routes import merchant_bp
from backend.api.user_routes import user_bp
from backend.api.crawl_routes import crawl_bp
from backend.api.customer_service_routes import cs_bp
from backend.docs.openapi import docs_bp

def create_app(config: AppConfig | None = None) -> Flask:
    app_config = config or AppConfig.from_env()
    setup_logging(app_config)

    app = Flask(__name__)
    app.config["APP_CONFIG"] = app_config
    app.config["UPLOAD_FOLDER"] = os.path.join(os.path.dirname(__file__), "uploads")
    os.makedirs(app.config["UPLOAD_FOLDER"], exist_ok=True)

    @app.get("/health")
    def health() -> tuple[dict[str, object], int]:
        return {
            "status": "ok",
            "service": "ecommerce-ai-copy-guide",
            "version": "0.2.0",
            "runtime": app_config.public_summary(),
        }, 200

    # ... 错误处理、CORS、上传文件路由 ...

    init_db()
    _create_default_users()

    app.register_blueprint(api_bp, url_prefix="/api")
    app.register_blueprint(auth_bp, url_prefix="/api")
    app.register_blueprint(merchant_bp, url_prefix="/api")
    app.register_blueprint(user_bp, url_prefix="/api")
    app.register_blueprint(crawl_bp, url_prefix="/api")
    app.register_blueprint(cs_bp, url_prefix="/api")
    app.register_blueprint(docs_bp)
    return app
```

### 1.1 各 Blueprint 职责划分

| Blueprint | 文件 | 前缀 | 职责 |
|---|---|---|---|
| `api_bp` | `backend/api/routes.py` | `/api` | 公共能力：商品浏览、AI 文案生成、智能导购、评论分析、直播脚本、统计数据等。 |
| `auth_bp` | `backend/api/auth_routes.py` | `/api` | 认证相关：登录、注册、找回密码、获取当前用户、更新资料、头像上传。 |
| `merchant_bp` | `backend/api/merchant_routes.py` | `/api` | 商家后台：商品 CRUD、知识库、问答统计、收入面板、订单管理、用户管理。 |
| `user_bp` | `backend/api/user_routes.py` | `/api` | 用户前台：购物车、订单、地址、收藏、浏览记录、RAG 问答、评论上传/提交。 |
| `crawl_bp` | `backend/api/crawl_routes.py` | `/api` | 爬虫管理：苏宁商品爬取、任务状态、向量索引构建。 |
| `cs_bp` | `backend/api/customer_service_routes.py` | `/api` | 客服消息：用户提问、AI 自动回复、商家会话列表与回复、未读数。 |
| `docs_bp` | `backend/docs/openapi.py` | 无 | API 文档（OpenAPI/Swagger）。 |

所有业务 Blueprint 统一挂载到 `/api` 前缀下，因此完整路径形如 `/api/products`、`/api/merchant/products`、`/api/user/cart`。

### 1.2 模型层与数据流

模型使用 SQLAlchemy 2.0 的 `Mapped`/`mapped_column` 风格定义，统一继承自 `backend.database.Base`。主要模型包括：

- `User` / `UserAddress` / `UserFavorite` / `BrowseHistory`（`backend/models/user.py`）
- `Product`（`backend/models/product.py`）
- `CartItem` / `Order` / `OrderItem`（`backend/models/shopping.py`）
- `CustomerServiceMessage`（`backend/models/customer_service.py`）
- `Review`（`backend/models/review.py`）

每个模型都提供 `to_dict()` 方法，用于序列化为 JSON 响应，避免直接暴露 ORM 对象。

## 二、认证与权限（require_auth / require_merchant）

认证逻辑集中在 `backend/services/auth_service.py`，采用基于 HMAC-SHA256 签名的简化 JWT Token，并在请求头 `Authorization: Bearer <token>` 中传递。

### 2.1 Token 结构与生成

```python
# backend/services/auth_service.py
JWT_SECRET = os.getenv("JWT_SECRET", "ecommerce-ai-secret-key-2026")
JWT_EXPIRE_HOURS = 24

def create_token(user_id: int, username: str, role: str) -> str:
    payload = {
        "user_id": user_id,
        "username": username,
        "role": role,
        "exp": int(time.time()) + JWT_EXPIRE_HOURS * 3600,
        "iat": int(time.time()),
    }
    payload_json = json.dumps(payload, separators=(",", ":"))
    payload_b64 = _base64_encode(payload_json)
    signature = _sign(f"{payload_b64}")
    return f"{payload_b64}.{signature}"

def verify_token(token: str) -> dict | None:
    try:
        parts = token.split(".")
        if len(parts) != 2:
            return None
        payload_b64, signature = parts
        expected_sig = _sign(payload_b64)
        if not hmac.compare_digest(signature, expected_sig):
            return None
        payload_json = _base64_decode(payload_b64)
        payload = json.loads(payload_json)
        if payload.get("exp", 0) < time.time():
            return None
        return payload
    except Exception as e:
        logger.debug(f"Token 验证失败: {e}")
        return None
```

Token 由 Base64 编码的 payload 与 HMAC 签名组成，中间用 `.` 分隔；校验时先验签名再验过期时间。该实现不依赖 `PyJWT`，轻量但功能完整。

### 2.2 密码哈希

```python
# backend/services/auth_service.py
def hash_password(password: str) -> str:
    salt = secrets.token_hex(16)
    hashed = hashlib.sha256(f"{salt}{password}".encode()).hexdigest()
    return f"{salt}${hashed}"

def verify_password(password: str, password_hash: str) -> bool:
    try:
        salt, stored_hash = password_hash.split("$")
        computed = hashlib.sha256(f"{salt}{password}".encode()).hexdigest()
        return hmac.compare_digest(computed, stored_hash)
    except (ValueError, AttributeError):
        return False
```

密码使用 16 字节随机 salt + SHA256 哈希，存储格式为 `salt$hash`，验证时使用 `hmac.compare_digest` 防止时序攻击。

### 2.3 认证装饰函数

```python
# backend/services/auth_service.py
def get_user_from_request(request) -> dict | None:
    auth_header = request.headers.get("Authorization", "")
    if auth_header.startswith("Bearer "):
        token = auth_header[7:]
        return verify_token(token)
    return None

def require_auth(request) -> tuple[dict | None, dict | None]:
    user = get_user_from_request(request)
    if not user:
        return None, {"error": "unauthorized", "message": "请先登录"}
    return user, None

def require_merchant(request) -> tuple[dict | None, dict | None]:
    user, error = require_auth(request)
    if error:
        return None, error
    if user.get("role") != "merchant":
        return None, {"error": "forbidden", "message": "需要商家权限"}
    return user, None
```

- `require_auth`：任意登录用户均可访问。
- `require_merchant`：在已登录基础上进一步校验 `role == "merchant"`，用于商家后台接口。

### 2.4 在路由中的使用方式

几乎每个受保护接口都遵循如下模式：先调用权限函数，出错直接返回 401/403，成功后再执行业务逻辑。

```python
# backend/api/merchant_routes.py
@merchant_bp.post("/merchant/products")
def create_product():
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401
    # ... 后续业务逻辑 ...

# backend/api/user_routes.py
@user_bp.get("/user/cart")
def get_cart():
    user_payload, error = require_auth(request)
    if error:
        return jsonify(error), 401
    # ... 后续业务逻辑 ...
```

登录接口本身不调用 `require_auth`，而是查库校验密码后签发 Token：

```python
# backend/api/auth_routes.py
@auth_bp.post("/auth/login")
def login():
    data = request.get_json(silent=True) or {}
    username = data.get("username", "").strip()
    password = data.get("password", "")
    role = data.get("role", "user")

    with SessionLocal() as db:
        user = db.execute(
            select(User).where(User.username == username)
        ).scalar_one_or_none()

        if not user:
            return jsonify({"error": "not_found", "message": "账号不存在"}), 404
        if not verify_password(password, user.password_hash):
            return jsonify({"error": "wrong_password", "message": "密码错误"}), 401
        if not user.is_active:
            return jsonify({"error": "disabled", "message": "账号已被禁用"}), 403
        if role and role != user.role:
            return jsonify({
                "error": "role_mismatch",
                "message": f"该账号是{'商家管理员' if user.role == 'merchant' else '普通用户'}，请切换身份登录"
            }), 403

        token = create_token(user.id, user.username, user.role)
        user_data = user.to_dict()
        user_data["token"] = token
        return jsonify({"message": "登录成功", "user": user_data, "token": token})
```

## 三、数据库会话管理（SessionLocal 使用）

数据库模块 `backend/database.py` 负责引擎创建、会话工厂、基类定义以及启动时的兼容性迁移。

### 3.1 数据库 URL 解析与引擎创建

```python
# backend/database.py
def _resolve_database_url() -> str:
    configured_url = os.getenv("DATABASE_URL", "")

    if not configured_url:
        return "sqlite:///./ecommerce_ai.db"

    if configured_url.startswith("sqlite"):
        return configured_url

    if configured_url.startswith("postgresql"):
        try:
            import psycopg2
            engine = create_engine(configured_url, pool_pre_ping=True)
            with engine.connect() as conn:
                conn.execute(__import__("sqlalchemy").text("SELECT 1"))
            return configured_url
        except ImportError:
            return "sqlite:///./ecommerce_ai.db"
        except Exception:
            return "sqlite:///./ecommerce_ai.db"

    return configured_url

DATABASE_URL = _resolve_database_url()

if DATABASE_URL.startswith("sqlite"):
    engine = create_engine(
        DATABASE_URL,
        echo=False,
        connect_args={"check_same_thread": False},
    )

    @event.listens_for(engine, "connect")
    def set_sqlite_pragma(dbapi_connection, connection_record):
        cursor = dbapi_connection.cursor()
        cursor.execute("PRAGMA foreign_keys=ON")
        cursor.close()
else:
    engine = create_engine(
        DATABASE_URL,
        pool_size=10,
        max_overflow=20,
        pool_pre_ping=True,
        echo=os.getenv("APP_ENV", "development") == "development",
    )
```

该设计支持环境变量 `DATABASE_URL` 动态切换 PostgreSQL 与 SQLite：当未配置或 PostgreSQL 不可用时，自动降级到本地 SQLite 文件 `./ecommerce_ai.db`，便于开发调试与部署迁移。

### 3.2 会话工厂与上下文管理

```python
# backend/database.py
SessionLocal = sessionmaker(bind=engine)

class Base(DeclarativeBase):
    pass

def get_db() -> Session:
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@contextmanager
def get_db_context() -> Session:
    db = SessionLocal()
    try:
        yield db
        db.commit()
    except Exception:
        db.rollback()
        raise
    finally:
        db.close()
```

- `SessionLocal` 是 SQLAlchemy 的 sessionmaker，绑定到上述引擎。
- `get_db()` 可作为生成器依赖，供未来接入 Flask-Injector 等框架使用。
- `get_db_context()` 是自动提交/回滚的上下文管理器，适用于需要自动事务边界的地方（如 `backend/api/routes.py` 的 AI 生成任务记录）。

### 3.3 路由中的典型用法

业务代码大量采用 `with SessionLocal() as db:` 显式管理会话：

```python
# backend/api/user_routes.py
@user_bp.get("/user/cart")
def get_cart():
    user_payload, error = require_auth(request)
    if error:
        return jsonify(error), 401

    try:
        with SessionLocal() as db:
            items = list(db.execute(
                select(CartItem).where(CartItem.user_id == user_payload["user_id"])
            ).scalars().all())
            # ... 组装响应 ...
    except Exception as e:
        logger.error(f"获取购物车失败: {e}", exc_info=True)
        return jsonify({"error": "server_error", "message": str(e)}), 500
```

在 `with SessionLocal() as db:` 块结束时，会话会自动关闭。对于写操作，通常显式调用 `db.commit()`；若发生异常，由外层 `try/except` 捕获并返回 500。

### 3.4 启动时兼容性迁移

`init_db()` 在应用启动时调用 `Base.metadata.create_all()` 建表，并执行一系列字段迁移函数，确保老数据库结构能平滑升级：

```python
# backend/database.py
def init_db() -> None:
    import backend.models
    Base.metadata.create_all(bind=engine)
    _add_password_plain_column()
    _add_product_media_columns()
    _add_review_media_columns()
    _add_product_published_column()
    _add_order_extra_columns()
    _add_display_id_columns()
    _add_review_user_id_column()
    _add_cs_sender_id_column()
    _hash_existing_plain_passwords()
    _normalize_order_numbers()
```

这些迁移函数通过 `sqlalchemy.inspect(engine)` 检查列是否存在，再用原生 `ALTER TABLE` 添加缺失列，并回填默认值（如 `is_published`、`display_id`、订单号规范化）。

## 四、商家端接口示例（商品管理、用户管理、订单管理）

商家端接口统一位于 `backend/api/merchant_routes.py`，全部通过 `require_merchant(request)` 校验权限。

### 4.1 商品管理

#### 4.1.1 创建商品

```python
# backend/api/merchant_routes.py
@merchant_bp.post("/merchant/products")
def create_product():
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401

    data = request.get_json(silent=True) or {}
    required = ["name"]
    for field in required:
        if not data.get(field):
            return jsonify({"error": "invalid_input", "message": f"缺少必填字段: {field}"}), 400

    image_urls = data.get("image_urls") or []
    videos = data.get("videos") or []
    image_url = data.get("image_url", "")
    if not image_url and image_urls:
        image_url = image_urls[0]

    with SessionLocal() as db:
        product = Product(
            name=data["name"],
            category=data.get("category", ""),
            price=float(data.get("price", 0) or 0),
            original_price=float(data["original_price"]) if data.get("original_price") else None,
            specs=data.get("specs", ""),
            selling_points=data.get("selling_points", ""),
            image_url=image_url,
            image_urls=image_urls if image_urls else None,
            videos=videos if videos else None,
            source_url=data.get("source_url", ""),
            platform="manual",
            brand=data.get("brand", ""),
            is_published=bool(data.get("is_published", True)),
        )
        db.add(product)
        product.display_id = generate_product_display_id(db)
        db.commit()
        db.refresh(product)

        product_data = product.to_dict()

        try:
            rag_service.auto_build_from_product(product)
        except Exception as e:
            logger.warning(f"知识库自动构建失败: {e}")

        return jsonify({"message": "商品创建成功", "product": product_data}), 201
```

流程说明：

1. 校验商家权限。
2. 校验必填字段 `name`。
3. 处理主图逻辑：若未传 `image_url` 但传了 `image_urls` 列表，则取第一张图作为主图。
4. 创建 `Product` 实例并写入数据库，同时调用 `generate_product_display_id(db)` 生成展示 ID。
5. 提交事务后，调用 `RAGService.auto_build_from_product()` 自动为商品构建知识库条目。

#### 4.1.2 更新与删除商品

```python
# backend/api/merchant_routes.py
@merchant_bp.put("/merchant/products/<int:product_id>")
def update_product(product_id: int):
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401

    data = request.get_json(silent=True) or {}
    with SessionLocal() as db:
        product = db.get(Product, product_id)
        if not product:
            return jsonify({"error": "not_found", "message": "商品不存在"}), 404

        for field in ["name", "category", "specs", "selling_points", "source_url"]:
            if field in data:
                setattr(product, field, data[field])
        if "is_published" in data:
            product.is_published = bool(data["is_published"])
        if "price" in data:
            product.price = float(data["price"])
        if "original_price" in data:
            product.original_price = float(data["original_price"]) or None

        if "image_urls" in data:
            product.image_urls = data["image_urls"] or None
        if "videos" in data:
            product.videos = data["videos"] or None
        if "image_url" in data:
            product.image_url = data["image_url"]
        elif "image_urls" in data and data["image_urls"]:
            product.image_url = data["image_urls"][0]

        db.commit()
        db.refresh(product)
        try:
            rag_service.auto_build_from_product(product)
        except Exception as e:
            logger.warning(f"知识库重建失败: {e}")

        return jsonify({"message": "商品更新成功", "product": product.to_dict()})
```

更新时同样保持 `image_url` 与 `image_urls` 第一张图一致，并在提交后重建知识库。

#### 4.1.3 商品列表查询

```python
# backend/api/merchant_routes.py
@merchant_bp.get("/merchant/products")
def list_merchant_products():
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401

    page = int(request.args.get("page", 1))
    page_size = int(request.args.get("page_size", 20))
    keyword = request.args.get("keyword", "")
    category = request.args.get("category", "")

    with SessionLocal() as db:
        from backend.repositories.product_repo import ProductRepository
        repo = ProductRepository(db)

        stmt = select(Product)
        if keyword:
            stmt = stmt.where(Product.name.contains(keyword))
        if category:
            stmt = stmt.where(Product.category == category)

        total = len(list(db.execute(stmt).scalars().all()))
        total_pages = max(1, (total + page_size - 1) // page_size)
        stmt = stmt.offset((page - 1) * page_size).limit(page_size)
        products = list(db.execute(stmt).scalars().all())

        all_categories = repo.list_categories()
        category_counts = {}
        for cat in all_categories:
            category_counts[cat] = repo.count(cat)

        published_count = len(list(db.execute(
            select(Product).where(Product.is_published == True)
        ).scalars().all()))

        return jsonify({
            "total": total,
            "published_count": published_count,
            "page": page,
            "page_size": page_size,
            "total_pages": total_pages,
            "categories": all_categories,
            "category_counts": category_counts,
            "products": [p.to_dict() for p in products],
        })
```

商家商品列表支持关键词、分类筛选与分页，同时返回全部分类及分类商品数、实际上架商品数，便于后台展示统计。

### 4.2 用户管理

#### 4.2.1 创建用户

```python
# backend/api/merchant_routes.py
@merchant_bp.post("/merchant/users")
def create_user():
    user, error = require_merchant(request)
    if error:
        return jsonify(error), 401

    data = request.get_json(silent=True) or {}
    username = data.get("username", "").strip()
    password = data.get("password", "")
    nickname = data.get("nickname", "").strip()
    role = data.get("role", "user")

    if not username or not password:
        return jsonify({"error": "invalid_input", "message": "用户名和密码不能为空"}), 400
    if len(password) < 6:
        return jsonify({"error": "weak_password", "message": "密码长度至少6位"}), 400
    if role not in ("user", "merchant"):
        return jsonify({"error": "invalid_role", "message": "角色必须是 user 或 merchant"}), 400

    with SessionLocal() as db:
        existing = db.execute(
            select(User).where(User.username == username)
        ).scalar_one_or_none()
        if existing:
            return jsonify({"error": "exists", "message": "用户名已存在"}), 409

        new_user = User(
            username=username,
            password_hash=hash_password(password),
            nickname=nickname or username,
            role=role,
        )
        db.add(new_user)
        db.commit()
        db.refresh(new_user)
        return jsonify({"message": "用户创建成功", "user": new_user.to_dict()}), 201
```

商家可创建普通用户或商家管理员账号，密码经 `hash_password` 处理，创建前检查用户名唯一性。

#### 4.2.2 删除用户

```python
# backend/api/merchant_routes.py
@merchant_bp.delete("/merchant/users/<int:user_id>")
def delete_user(user_id: int):
    merchant, error = require_merchant(request)
    if error:
        return jsonify(error), 401

    if user_id == merchant["user_id"]:
        return jsonify({"error": "forbidden", "message": "不能删除自己的账号"}), 403

    with SessionLocal() as db:
        target_user = db.get(User, user_id)
        if not target_user:
            return jsonify({"error": "not_found", "message": "用户不存在"}), 404

        db.execute(delete(CartItem).where(CartItem.user_id == user_id))
        db.execute(delete(UserFavorite).where(UserFavorite.user_id == user_id))
        db.execute(delete(BrowseHistory).where(BrowseHistory.user_id == user_id))
        db.execute(delete(UserAddress).where(UserAddress.user_id == user_id))
        db.execute(delete(CustomerServiceMessage).where(CustomerServiceMessage.user_id == user_id))
        db.execute(update(QARecord).where(QARecord.user_id == user_id).values(user_id=None))
        db.execute(update(Review).where(Review.user_id == user_id).values(user_id=None))

        order_ids = [
            row[0] for row in db.execute(
                select(Order.id).where(Order.user_id == user_id)
            ).all()
        ]
        if order_ids:
            db.execute(delete(OrderItem).where(OrderItem.order_id.in_(order_ids)))
            db.execute(delete(Order).where(Order.id.in_(order_ids)))

        db.delete(target_user)
        db.commit()
        return jsonify({"message": "用户已删除"})
```

删除用户前会级联清理购物车、收藏、浏览记录、地址、客服消息、问答记录、评论关联以及订单数据，避免外键约束冲突或遗留脏数据。

                    filters.append(user_filter)

### 6.2 全局错误处理

Flask 应用层注册了 404、500、400 的全局错误处理器：

```python
# backend/app.py
@app.errorhandler(404)
def not_found(_error: Exception):
    logger.warning(f"404 Not Found: {_error}")
    return jsonify({
        "error": "not_found",
        "message": "请求的资源不存在"
    }), 404

@app.errorhandler(500)
def server_error(_error: Exception):
    logger.error(f"500 Internal Server Error: {_error}", exc_info=True)
    return jsonify({
        "error": "server_error",
        "message": "服务器内部错误"
    }), 500

@app.errorhandler(400)
def bad_request(_error: Exception):
    logger.warning(f"400 Bad Request: {_error}")
    return jsonify({
        "error": "bad_request",
        "message": "请求格式错误"
    }), 400
```

- 400/404 记录为 `warning`。
- 500 记录为 `error` 并附带堆栈（`exc_info=True`），但对外返回统一的“服务器内部错误”，避免泄露敏感信息。

## 七、小结

本项目后端采用清晰的模块化设计：

- **入口层**：`app.py` 负责应用装配、全局中间件、Blueprint 注册与启动初始化。
- **路由层**：按业务领域划分为多个 Blueprint，职责边界清晰。
- **服务层**：`auth_service.py` 提供统一认证与权限校验；`RAGService`、`FileUploadService`、`AIProvider` 等处理复杂业务。
- **数据层**：SQLAlchemy 模型 + Repository 模式 + Redis 缓存，兼顾开发效率与性能。
- **运维友好**：SQLite/PostgreSQL 自动降级、启动时自动迁移、统一日志与错误处理。

整体代码风格一致，所有受保护接口均先校验身份，再执行业务；数据库会话采用 `with SessionLocal() as db:` 显式管理，写操作显式 `commit`，异常统一捕获并记录，适合后续扩展为正式生产服务。

## 八、近期接口修复

### 8.1 商家回复用户接口 HTTP 500

`backend/api/customer_service_routes.py` 的 `merchant_reply` 原代码将 `require_merchant` 返回的 JWT payload 字典当作 `User` 对象使用，访问 `merchant.id` 时抛出 `AttributeError`，最终表现为前端收到 HTTP 500。

修复后，接口先从 payload 中取出 `user_id`，再查询 `User` 表获取商家对象，并用该对象的 `id` 写入 `sender_id`：

```python
# backend/api/customer_service_routes.py
merchant_user = db.get(User, merchant["user_id"])
if not merchant_user:
    return jsonify({"error": "not_found", "message": "商家账号不存在"}), 404

msg = CustomerServiceMessage(
    user_id=user_id,
    product_id=product_id,
    sender_id=merchant_user.id,
    sender_role="merchant",
    content=content,
    is_read=True,
)
db.add(msg)
db.commit()
db.refresh(msg)

return jsonify({
    "message": "回复成功",
    "data": _message_to_dict(msg, {user_id: user}, {merchant_user.id: merchant_user}, product_map),
}), 201
```

`_message_to_dict` 随后通过 `sender_id` 到 `merchant_map` 查找商家真实昵称与头像，保证前端会话列表与详情页都能正确展示回复者信息。

### 8.2 商家删除用户接口

`backend/api/merchant_routes.py` 的 `delete_user` 面临外键约束问题。若用户存在购物车、收藏、地址、订单、评论、客服消息等关联数据，直接 `db.delete(target_user)` 会导致数据库报错。

修复方案是按业务依赖顺序先清理或脱敏关联数据，再删除用户：

```python
# backend/api/merchant_routes.py
# 1. 清理弱关联数据
db.execute(delete(CartItem).where(CartItem.user_id == user_id))
db.execute(delete(UserFavorite).where(UserFavorite.user_id == user_id))
db.execute(delete(BrowseHistory).where(BrowseHistory.user_id == user_id))
db.execute(delete(UserAddress).where(UserAddress.user_id == user_id))
db.execute(delete(CustomerServiceMessage).where(CustomerServiceMessage.user_id == user_id))

# 2. 对需要保留历史记录但需脱敏的表置空 user_id
db.execute(update(QARecord).where(QARecord.user_id == user_id).values(user_id=None))
db.execute(update(Review).where(Review.user_id == user_id).values(user_id=None))

# 3. 清理强关联的订单及订单商品
order_ids = [row[0] for row in db.execute(select(Order.id).where(Order.user_id == user_id)).all()]
if order_ids:
    db.execute(delete(OrderItem).where(OrderItem.order_id.in_(order_ids)))
    db.execute(delete(Order).where(Order.id.in_(order_ids)))

# 4. 删除用户本体
db.delete(target_user)
db.commit()
```

同时接口保留自保护：

```python
if user_id == merchant["user_id"]:
    return jsonify({"error": "forbidden", "message": "不能删除自己的账号"}), 403
```
